Now let's assemble the full view. Watch how your two lines just drop into the 1-2-3 shape:

from django.shortcuts import render
from .models import WeatherReading


def dashboard(request):
    # ── Step 1: get the data ──
    latest = WeatherReading.objects.order_by('-recorded_at').first()   # headline (1 row)
    readings = WeatherReading.objects.order_by('-recorded_at')[:24]    # table (24 rows)

    # ── Step 2: pack the bag (context) ──
    context = {
        'latest': latest,
        'readings': readings,
    }

    # ── Step 3: render ──
    return render(request, 'weather/dashboard.html', context)

That's the whole thing. Nothing new — the exact shape from Phase 1/2, just with your two ORM lines feeding it.

Read the context bag carefully — those labels ('latest', 'readings') are the names the template will use. When we wire the HTML next, {{ latest.city }} and {% for reading in readings %} will reach into this bag. That's the handoff.
