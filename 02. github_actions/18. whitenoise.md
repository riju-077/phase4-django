# NOTES 09 — whitenoise (keeping the admin page styled)

**Date:** 2026-07-18

## The one-line goal

Make sure the **admin page** still looks styled after we turn `DEBUG=False`.

---

## First, what are "static files"?

Static files = the **CSS, JavaScript, and images** that make a page look nice.
They don't change per user — they're the same for everyone. Hence "static".

---

## Which page does this even affect?

Only the **admin page** (`/admin/`) — Django's ready-made page.

- **Your dashboard page** got its look from **Google Stitch**. That styling loads
  from the **internet** (a link inside the HTML). So the dashboard is fine no matter what.
- **The admin page** gets its look from **Django's own CSS files**. THAT is the part
  we have to take care of.

So this whole note is only about keeping the **admin** looking correct.

---

## Why does the admin break when DEBUG=False?

Right now (`DEBUG=True`), Django does you a **favour**: it hands out the admin's
CSS files itself.

When you set `DEBUG=False`, Django **stops doing that favour** — on purpose.
Its thinking: *"On a real server, handing out CSS is not my job. A proper tool
should do it."*

So the admin asks for its CSS → **nobody hands it over** → the page loads as
plain, unstyled text.

---

## Then why not just keep DEBUG=True forever?

Because `DEBUG=True` is **dangerous** on a real server:

1. **Strangers see your code.** On an error, `DEBUG=True` shows the full error page
   — your code, file paths, bits of settings. Fine on your laptop, a gift to an
   attacker on a live site.
2. **It slowly eats memory.** With `DEBUG=True`, Django remembers every database
   query forever. A server that runs for weeks keeps piling them up until it crashes.

So `DEBUG=True` is **development-only**. We must be able to turn it off — which
means we need the "proper tool" for CSS. That tool is **whitenoise**.

---

## What whitenoise is

**whitenoise** is a small helper that takes over the job Django drops.
It lets your app (running under **gunicorn** in the container) hand out static
files correctly, even with `DEBUG=False`.

---

## What we changed

### 1. Installed it
```
uv add whitenoise
```

### 2. `settings.py` — added it to the middleware list

**BEFORE**
```python
MIDDLEWARE = [
    "django.middleware.security.SecurityMiddleware",
    "debug_toolbar.middleware.DebugToolbarMiddleware",
    ...
]
```

**AFTER**
```python
MIDDLEWARE = [
    "django.middleware.security.SecurityMiddleware",
    "whitenoise.middleware.WhiteNoiseMiddleware",   # ← added
    "debug_toolbar.middleware.DebugToolbarMiddleware",
    ...
]
```
**Why here?** Middleware is a line of checkpoints every request passes through.
whitenoise must sit **right after** the security checkpoint — that's what its
docs say. It catches requests asking for a CSS/JS file and hands the file back.

### 3. `settings.py` — told it WHERE the files live

**ADDED** under `STATIC_URL`:
```python
STATIC_ROOT = BASE_DIR / "staticfiles"

STORAGES = {
    "default": {
        "BACKEND": "django.core.files.storage.FileSystemStorage",
    },
    "staticfiles": {
        "BACKEND": "whitenoise.storage.CompressedManifestStaticFilesStorage",
    },
}
```
- `STATIC_ROOT` = the **one folder** where all static files get gathered.
- `STORAGES` = tells Django to use whitenoise's smart storage, which **compresses**
  the files (smaller = faster to send).

### 4. Gathered the files into that folder
```
python manage.py collectstatic
```
Result: **164 static files copied**, 470 post-processed.

"Gathered" = Django went through every app (admin, etc.), found their CSS/JS, and
copied them all into the single `staticfiles/` folder for whitenoise to serve.

### 5. Ignored the generated folder

`staticfiles/` is **auto-generated** by `collectstatic` — it can be rebuilt anytime.
So we added it to `.gitignore` (never commit 164 machine-made files).

---

## The mental picture

```
collectstatic  →  copies all CSS/JS  →  into  staticfiles/  folder
                                             │
request for a CSS file  →  whitenoise middleware  →  reads from staticfiles/  →  sends it back
                                                     (works even with DEBUG=False)
```

---

## Order matters (a gotcha)

Install whitenoise **before** running any Django command. Our settings now point
to `whitenoise.storage`, so if whitenoise isn't installed yet, Django can't even
start. Install first, then collectstatic.
