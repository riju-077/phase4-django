Q2: What is ghcr.io and why does it exist?

ghcr.io = GitHub Container Registry. A registry is a shelf that images live on, with an address.

You've already used one. This line:

image: postgres:16

Docker didn't have Postgres. It went to a shelf on the internet — Docker Hub — found postgres:16, downloaded it. That's a registry.

ghcr.io is the same idea, run by GitHub.

Why do we need a shelf at all? Because an image is a big binary file — hundreds of megabytes. You can't put that in git (git is for text you edit). You can't email it.

▎ Git = the shelf for your code.
▎ A registry = the shelf for your built image.
▎
▎ Two different things, two different shelves.

Why ghcr.io and not Docker Hub? Your code already lives on GitHub. ghcr.io sits right next to it, free, same account, and CI can push to it with a password GitHub hands out automatically. Less setup.
