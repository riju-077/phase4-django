● Here's the thumb rule:

  ▎ Add a package when it does something hard, risky, or boring better than you could yourself. Avoid it when you'd only be saving a few lines 
  ▎ you could easily write and own.

  That's it. One sentence.

  How to use it in 5 seconds

  Ask yourself two questions:

  1. "Could I safely write this myself in a few lines?"
    - Yes → don't add a package.
    - No (it's complex/security-sensitive/lots of edge cases) → add it.
  2. "Is it popular and actively maintained?"
    - Yes → safe to add.
    - No → avoid, find an alternative.

  If question 1 says "I shouldn't write this myself" and question 2 says "it's trusted" → add it without guilt.

  Applied to pyotp

  - Could I write OTP myself safely? No — it's security crypto, easy to get wrong. ✅ add it
  - Is it trusted? Yes — industry standard. ✅ add it

  → Add it. Easy call.
