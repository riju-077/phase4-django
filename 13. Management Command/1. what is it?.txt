Chunk 1 — Management command

First the why, super simple:

You already use commands like these every day:

python manage.py runserver
python manage.py migrate
python manage.py createsuperuser

Those are built-in commands Django gave you.

A management command is just your own custom one — a command you write, that you can run the same way:

python manage.py fetch_all_cities

That's the whole idea: a button you can press from the terminal to run your own code.

Got that before I show you why we'd want one?

The why, one concrete example:

Right now your app fetches London only — triggered by a web click (/fetch/) or the Beat bot.

But imagine you want to fetch many cities at once — London, New York, Tokyo — all in one go, by hand, without opening a browser or clicking anything.

That's the perfect job for a management command:

python manage.py fetch_all_cities

One line in the terminal → fires off fetches for all your cities at once.

Why not just a web page? Because this is a back-office batch job, not something a customer should ever see or trigger. Terminal is exactly where it belongs.
