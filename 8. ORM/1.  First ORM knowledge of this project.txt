Yes — you've got it exactly right, and that SQL is spot on. ✅ Look at your result: London, 24.2°, newest row. That's precisely the "headline weather" the view needs.

Let me answer your three things:

1. "ORM = SQL in a Pythonic way?" — Yes, exactly. You write Python, and Django translates it into SQL behind your back and runs it. Same query, different language on the surface. (Later, the Debug Toolbar will literally show you the SQL each ORM line generated — you'll see the translation with your own eyes. Great moment coming.)

2. "Is there a dedicated site to learn ORM = SQL?" — Yes. The official one:
👉 QuerySet API reference — https://docs.djangoproject.com/en/5.0/ref/models/querysets/
That's the dictionary that maps every SQL idea to its ORM method. Bookmark it.

---
3. Now the fun part — since you know SQL, you already know the ORM. Watch the translation, clause by clause:

Your SQL:
SELECT ... FROM weather_weatherreading   ORDER BY recorded_at DESC   LIMIT 1;

The ORM, piece by piece:

┌─────────────────────────────┬──────────────────────────────────────────────┐
│          Your SQL           │                ORM equivalent                │
├─────────────────────────────┼──────────────────────────────────────────────┤
│ FROM weather_weatherreading │ WeatherReading.objects                       │
├─────────────────────────────┼──────────────────────────────────────────────┤
│ ORDER BY recorded_at DESC   │ .order_by('-recorded_at') ← the - means DESC │
├─────────────────────────────┼──────────────────────────────────────────────┤
│ LIMIT 1 (grab one row)      │ .first()                                     │
└─────────────────────────────┴──────────────────────────────────────────────┘

Chain them left to right, same order as SQL reads:

WeatherReading.objects.order_by('-recorded_at').first()

That single line = your entire SQL query. .first() returns one object (the newest reading), or None if the table is empty.

---
Bonus — a shortcut. Remember your model's Meta? You already set:
ordering = ['-recorded_at']
That means the ORM already sorts by newest by default. So you could even drop the order_by and just write:
WeatherReading.objects.first()
Both work. I'd keep the explicit .order_by('-recorded_at') for now — it's clearer to a reader why .first() gives the newest.

---
So there's half our view — the headline reading:

latest = WeatherReading.objects.order_by('-recorded_at').first()

Next tiny step: the list for the table rows. In SQL you'd drop the LIMIT 1 and maybe LIMIT 24 instead. Want to try writing that ORM line yourself, using the same table above? (Hint: instead of .first(), there's a method that grabs a slice of rows.)
